Wenn diese Option auf true gesetzt ist, führt das Plug-in nur Abfragen durch, akzeptiert oder lädt jedoch nicht den Repository-Arbeitsbereich.

Anmerkung: Wenn Sie diese Option auswählen, müssen Sie sicherstellen, dass Änderungen innerhalb des nächsten Abfrageintervalls im Arbeitsbereich als Teil des Builds akzeptiert werden.

Wenn diese Option auf true gesetzt ist, fragt RTCScm nur den Repository-Arbeitsbereich ab und führt die folgenden Aktionen nicht aus:

  • Akzeptieren
  • load
  • Änderungsprotokoll im Build aufzeichnen
  • Jenkins-Build mit für Plug-in spezifischen Daten aktualisieren

Weiterentwicklung und Abwärtskompatibilität

  • Für Jobs, die vor 2.3.0 erstellt wurden, hat die Option den Wert false.
  • Bei neuen Jobs hat die Option den Wert false.

Vorgaben

  • Wird für Pipelinejobs unterstützt.
  • Wird für Builddefinitionen und Buildarbeitsbereichstypen unterstützt.
