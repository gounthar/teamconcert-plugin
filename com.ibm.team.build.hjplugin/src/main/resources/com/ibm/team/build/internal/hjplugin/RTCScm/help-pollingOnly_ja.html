このオプションがtrueに設定されている場合、プラグインはポーリングのみを実行しますが、リポジトリー・ワークスペースを受け入れることもロードすることもしません。

注: このオプションを選択する場合は、次のポーリング間隔内に、ビルドの一部としてワークスペースへの変更が受け入れられるようにする必要があります。

このオプションが true に設定されている場合、RTCScm はリポジトリー・ワークスペースのみをポーリングし、以下のアクションは実行しません。

  • 受諾
  • ロード
  • ビルドでの変更ログの記録
  • プラグイン固有のデータによる Jenkins ビルドの更新

進化と後方互換性

  • 2.3.0より前に作成されたジョブの場合、オプションの値は false になります。
  • 新規ジョブの場合、オプションの値は false になります。

制約

  • パイプライン・ジョブでサポートされます。
  • ビルド定義およびビルド・ワークスペース・タイプでサポートされます。
