Lorsque cette option est définie sur true, le plug-in effectue uniquement une interrogation mais n'accepte pas ou ne charge pas l'espace de travail de référentiel.

Remarque : si vous sélectionnez cette option, vous devez vous assurer que les modifications sont acceptées dans l'espace de travail dans le cadre de la génération, au cours de l'intervalle d'interrogation suivant.

Si cette option est définie sur true, RTCScm interroge uniquement l'espace de travail de référentiel et n'effectue pas les actions suivantes :

  • accepter
  • load
  • enregistrement du journal des modifications dans la génération
  • mise à jour de la génération Jenkins avec des données spécifiques au plug-in

Evolution et compatibilité avec les versions antérieures

  • Pour les travaux créés avant la version 2.3.0, la valeur de l'option sera false.
  • Pour les nouveaux travaux, la valeur de l'option sera false.

Contraintes

  • Pris en charge pour les travaux de pipeline.
  • Pris en charge pour les types de définition de génération et d'espace de travail de génération.
