I log e le risorse utente possono essere raggruppati in un componente nel risultato della build. Fornendo un nome componente, è possibile identificare in modo univoco un log o una risorsa utente, soprattutto quando sono presenti più log o risorse utente con lo stesso nome file tra componenti differenti ma non all'interno dello stesso componente.

Se sono presenti più log o risorse utente con lo stesso nome file in un componente, utilizzare il campo ID contenuto per identificare in modo univoco il contributo.

Se si lascia vuoto questo campo, verrà richiamato il primo log o risorsa utente che corrisponde al nome file.
